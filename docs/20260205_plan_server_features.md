### 제안 내용 (juyoung)

> 이 서버는 소소한 힐링 서버야. 자유롭게 건축도 하고 낚시도 하고 농사도 짓고 광물도 캐고 등등 할 수 있어야 해. 그러면서 우리 서버만의 커스텀 생선, 작물, 광물등도 추가할거고. 당장은 아니어도 추후 화폐도 추가했으면 좋겠어. 플레이어 규모는 최대 10명 언저리일 것 같아.

**요약:**
- 컨셉: 소소한 힐링 서버
- 활동: 건축, 낚시, 농사, 채굴 등 자유로운 플레이
- 커스텀 컨텐츠: 생선, 작물, 광물
- 추후 추가: 화폐 시스템
- 규모: 최대 10명

### 리뷰 (Claude)

**컨셉에 대해:**
힐링 서버 + 커스텀 컨텐츠 조합은 좋은 방향입니다. 바닐라 마인크래프트의 감성을 유지하면서 "우리 서버만의 특별함"을 더할 수 있어요.

**커스텀 컨텐츠 의견:**

| 컨텐츠 | 구현 난이도 | 코멘트 |
|--------|-------------|--------|
| 커스텀 생선 | ★★☆ | 낚시 시스템에 훅을 걸어 새 아이템 드롭. 비교적 간단 |
| 커스텀 작물 | ★★★ | 새 블록(작물) + 성장 단계 + 수확물. 텍스처 필요 |
| 커스텀 광물 | ★★★ | 새 블록 + 월드젠(생성 규칙) + 드롭 아이템. 텍스처 필요 |
| 화폐 시스템 | ★★★★ | 데이터 저장, 명령어, UI 등 복잡. 후순위 적절 |

**질문:**
1. 커스텀 아이템들의 **용도**가 있나요? (예: 커스텀 생선으로 특별한 음식 제작, 광물로 특수 도구 제작 등) 아니면 수집/판매용?
2. **텍스처/모델**은 직접 제작하시나요? 아니면 기존 텍스처 재활용?
3. 1차로 어떤 컨텐츠부터 시작하고 싶으세요?

### Q1. 커스텀 아이템 용도

**juyoung:**
> 수집/판매용이면서 동시에 음식/도구 제작도 추가되면 좋을 것 같아.

**Claude 리뷰:**

좋은 방향입니다. 이렇게 하면 아이템에 **여러 가치**가 생겨요:

| 용도 | 예시 | 플레이어 동기 |
|------|------|---------------|
| 수집 | 도감 채우기, 희귀템 모으기 | 완성욕, 성취감 |
| 판매 | 화폐로 교환 | 경제 활동 |
| 제작 재료 | 음식/도구 크래프팅 | 실용적 가치 |

**구현 관점:**

```
커스텀 생선 → 구워서 특수 음식 (버프?)
커스텀 작물 → 특수 음식 or 포션 재료
커스텀 광물 → 특수 도구/방어구 or 장식 블록
```

**추가 질문:**
- 특수 음식/도구에 **바닐라에 없는 효과**(새로운 버프, 특수 능력 등)를 넣을까요?
- 아니면 바닐라 스타일로 **스탯만 다르게** (더 좋은 포만감, 더 높은 내구도 등)?

**juyoung:**
> 둘 다 섞으면 더 좋을 것 같은데 가능하겠어?

**Claude 리뷰:**

네, **충분히 가능**합니다!

오히려 섞는 게 더 자연스러워요. 아이템 **등급/희귀도**에 따라 나누면 밸런스도 잡기 좋습니다:

| 등급 | 효과 타입 | 예시 |
|------|-----------|------|
| 일반 | 바닐라 스타일 | 포만감 좋은 생선구이, 내구도 높은 곡괭이 |
| 희귀 | 새로운 효과 | 낚시 속도 버프 음식, 광물 2배 드롭 곡괭이 |
| 특별 | 유니크 효과 | 물속 호흡 영구 음식, 자동 제련 곡괭이 등 |

**구현 난이도:**
- 바닐라 스타일: ★★☆ (기존 API 활용)
- 새로운 효과: ★★★~★★★★ (커스텀 Effect/Enchantment 구현)

**제안:**
1차 구현에서는 **일반 등급(바닐라 스타일)** 먼저 만들고, 이후 희귀/특별 등급을 추가하는 방식은 어떨까요? 점진적으로 확장하기 좋습니다.

→ **합의됨**: 등급별로 효과 차등화, 1차는 바닐라 스타일부터

### Q2. 텍스처/모델

**juyoung:**
> 텍스처/모델은 일단 기존 것을 재활용 한 뒤에 변경 가능해?

**Claude 리뷰:**

**네, 완전히 가능합니다!**

텍스처는 그냥 PNG 파일이라서 언제든 교체할 수 있어요:

```
src/main/resources/assets/estherserver/textures/item/
├── custom_fish.png      ← 나중에 이 파일만 교체하면 됨
├── custom_crop.png
└── custom_ore.png
```

**개발 단계별 접근:**

| 단계 | 텍스처 | 목적 |
|------|--------|------|
| 1차 | 바닐라 재활용/색조 변경 | 기능 구현에 집중 |
| 2차 | 커스텀 텍스처 적용 | 비주얼 완성 |

**재활용 방법:**
1. **바닐라 텍스처 그대로** - 예: 대구 텍스처를 커스텀 생선에 임시 사용
2. **색조(Tint) 변경** - 코드로 색상만 바꿔서 구분
3. **간단한 편집** - 바닐라 텍스처 살짝 수정

기능 먼저 완성하고 텍스처는 나중에 다듬는 방식, 추천합니다!

→ **합의됨**: 바닐라 텍스처 재활용 후 추후 교체

### Q3. 구현 순서

**juyoung:**
> 구현은 전부 너가 할 거기 때문에 너가 구현 난이도에 맞춰 순서를 정해봐.

**Claude 제안:**

구현 난이도 순으로 정리하면:

| 순서 | 컨텐츠 | 난이도 | 이유 |
|------|--------|--------|------|
| 1차 | 커스텀 생선 | ★★☆ | 새 블록 불필요, 낚시 루트테이블에 추가만 하면 됨 |
| 2차 | 커스텀 작물 | ★★★ | 새 블록 필요, 성장 단계 구현, 하지만 월드젠 불필요 |
| 3차 | 커스텀 광물 | ★★★ | 새 블록 + 월드젠(어디에 생성될지) 구현 필요 |

**상세 이유:**

**1. 커스텀 생선 (가장 쉬움)**
- 아이템만 추가하면 됨 (블록 X)
- 낚시 루트테이블에 훅
- 구운 생선 = 기존 음식 시스템 활용
- 빠르게 결과물 확인 가능 → 모드 구조 익히기 좋음

**2. 커스텀 작물 (중간)**
- 씨앗 아이템 + 작물 블록 필요
- 성장 단계 (1~7단계) 구현
- 수확물 아이템
- 월드젠은 불필요 (플레이어가 직접 심으니까)

**3. 커스텀 광물 (가장 복잡)**
- 광석 블록 + 드롭 아이템
- **월드젠** 필요 (Y좌표, 생성 빈도, 바이옴 등)
- 제련 레시피
- 도구/방어구까지 확장 시 추가 작업

**이 순서로 진행하면:**
- 점진적으로 난이도 상승
- 각 단계에서 배운 걸 다음에 활용
- 중간중간 플레이 가능한 결과물 확인

→ **합의됨**: 생선 → 작물 → 광물 순서로 구현

### 결정된 사항

#### 서버 컨셉
- **소소한 힐링 서버**
- 자유로운 플레이: 건축, 낚시, 농사, 채굴
- 플레이어 규모: 최대 10명

#### 커스텀 컨텐츠
| 컨텐츠 | 용도 | 구현 순서 |
|--------|------|-----------|
| 커스텀 생선 | 수집/판매/음식 제작 | 1차 |
| 커스텀 작물 | 수집/판매/음식 제작 | 2차 |
| 커스텀 광물 | 수집/판매/도구 제작 | 3차 |

#### 아이템 효과 방향
- **등급별 차등화**: 일반(바닐라 스타일) → 희귀(새 효과) → 특별(유니크)
- 1차 구현은 바닐라 스타일부터, 이후 점진적 확장

#### 텍스처/모델
- 1차: 바닐라 텍스처 재활용
- 2차: 커스텀 텍스처로 교체

#### 추후 추가 예정
- 화폐 시스템
- 상점 시스템
