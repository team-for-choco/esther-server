# 타 마인크래프트 서버 분석 및 기능 벤치마킹

## 작업 개요
다른 마인크래프트 서버들의 주요 기능을 분석하고, Esther Server에 도입할 만한 기능을 검토한다.

## 작업 전 요청사항
- Esther Server 컨셉(소소한 힐링 서버, 10명 규모)에 맞는 서버들을 대상으로 분석
- PvP/경쟁 서버 제외, 커뮤니티/생활/힐링 서버 중심

## 10명 소규모 서버에 특히 효과적인 기능 TOP 5

1. **도감 + 희귀도 시스템** — 띵타이쿤의 핵심 동력. 도감 달성이 기능 해금/보상과 연동되어 장기 플레이 유도. 악어놀이터에서도 도감 수집 시 스탯 상승.
2. **요리 시스템** — 농사+낚시 결과물을 요리로 연결하는 콘텐츠 순환. 띵타이쿤/악어놀이터 모두 핵심 경제 활동.
3. **플레이어 상점 (쇼핑 디스트릭트)** — 각자 전문 분야를 갖고 상점 운영. 자연스러운 상호의존성.
4. **일일/주간 퀘스트** — 매일 접속할 이유 제공. 소규모 서버에서 접속 유지가 가장 큰 과제.
5. **작물/물고기 등급 시스템** — 악어놀이터의 은별/금별처럼 같은 아이템도 품질 차이. 수집+거래 모두에 깊이 추가.

### 구현 로드맵 (추천 순서)

개발 의존성과 난이도를 고려한 순서. QoL 기능은 대형 시스템 사이에 배치하여 부담 분산.

#### Phase 1: 콘텐츠 기반 확장
> 현재 생선/작물이 있으니, 광물을 추가하고 아이템 등급 시스템의 기반을 만든다.

| 순서 | 기능 | 의존성 | 난이도 | 비고 |
|------|------|--------|--------|------|
| 1-1 | 루트 모디파이어 replace 방식 | 없음 | ★☆☆ | 기존 예정 작업 (생선 드롭 대체) |
| 1-2 | 커스텀 광물 + 월드젠 | 없음 | ★★★ | 기존 계획 (CHOCO-75 이후 3차) |

### 로드맵 요약

```
[현재] 생선 ✅ → 작물 ✅ → replace 수정 → 광물 → 등급 시스템 → 한국식 작물
                                              ↓
                              QoL (슬리핑, 낮 연장, 앉기)
                                              ↓
                                 요리 시스템 → 도감 시스템
                                              ↓
                                     QoL (땅 보호)
                                              ↓
                              화폐 → 플레이어 상점 → NPC 상인
                                              ↓
                                 전문 분야 보너스 → 스킬 트리
```

### 순서 결정 근거

1. **등급 시스템을 일찍 도입** — 이후 추가되는 모든 아이템(한국식 작물, 요리 등)이 처음부터 등급을 가질 수 있음. 나중에 도입하면 기존 아이템 전부 마이그레이션 필요.
2. **요리 → 도감 순서** — 도감에 넣을 수집 대상이 충분해야 도감이 의미 있음. 요리가 추가되면 생선+작물+광물+요리로 도감 볼륨 확보.
3. **경제는 Phase 3** — 거래할 컨텐츠가 다양해진 후에 경제를 열어야 상점이 활성화됨.
4. **QoL은 사이사이 배치** — 대형 시스템 개발 사이에 가벼운 QoL 작업으로 번아웃 방지 + 즉각적 체감 개선.
