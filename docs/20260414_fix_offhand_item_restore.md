# 타인 토지 블록 설치 취소 시 보조 손 아이템 미반환 수정

## 작업 개요
CHOCO-147에서 수정한 타인 토지 블록 설치 취소 시 아이템 복원 로직이 클라이언트 동기화만 수행하고, 서버 측 아이템 수량을 실제로 복원하지 않는 문제를 수정한다. 특히 보조 손(off-hand)으로 설치 시 아이템이 사라지는 문제가 재발함.

## 작업 전 요청사항
- CHOCO-147 수정 이후에도 보조 손에 든 아이템으로 타인 토지에 설치 시 아이템이 사라짐

## 작업 진행 중 결정된 사항등, 작업관련 주요 정보

### 원인 분석
기존 `restorePlacedItem()`은 다음 코드만 실행:
```kotlin
player.containerMenu.broadcastChanges()
player.inventoryMenu.sendAllDataToRemote()
```
이는 서버의 현재 인벤토리 상태를 클라이언트에 전송할 뿐, 서버에서 이미 차감된 아이템을 복원하지 않는다. NeoForge의 `EntityPlaceEvent` 취소는 블록 상태만 되돌리고 아이템 수량은 복원하지 않으므로, 서버 측에서 직접 아이템을 돌려줘야 한다.

### 수정 내용
`restorePlacedItem()`에서 설치된 블록의 아이템을 주 손/보조 손 모두 확인하여 서버 측에서 수량을 직접 복원:
1. 주 손에 해당 아이템이 있으면 → `grow(1)`
2. 보조 손에 해당 아이템이 있으면 → `grow(1)`
3. 주 손이 비어있으면 (마지막 1개 설치한 경우) → 주 손에 새 아이템 설정
4. 보조 손이 비어있으면 → 보조 손에 새 아이템 설정
5. 모두 해당 없으면 → 인벤토리에 추가, 불가하면 월드에 드롭
