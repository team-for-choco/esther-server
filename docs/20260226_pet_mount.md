# 펫(탈것) 시스템 테스트 구현

## 작업 개요
펫 보관함 GUI에서 펫을 선택하면 플레이어가 탑승한 상태로 소환되는 탈것 시스템.
등급에 따라 속도가 달라지며, 테스트로 일반(Common) 등급 고양이 펫 1종만 우선 구현.
코드 기반 엔티티 모델 사용 (추후 GeckoLib 등으로 교체 가능).

CHOCO-119

## 작업 전 요청사항
- 고양이 모델: 코드 기반 사족보행 모델 (몸통 + 머리 + 다리 4개 + 꼬리)
- 텍스처: 플레이스홀더 16x16 (추후 교체)
- 속도: 스프린트(0.13)보다 ~8% 빠른 0.14

## 작업 진행 중 결정된 사항등, 작업관련 주요 정보
- PetEntity는 PathfinderMob 기반 (탑승 조종 API 활용)
- PlayerRideableJumping 구현으로 점프 지원 (jumpPower 0~100 → 0.42~1.0 스케일)
- 하차 시 자동 디스폰, 저장 안 됨
- H키로 보관함 열기 (P키는 바닐라 사회적 상호작용과 충돌하여 변경)
- 플레이어에게 기본 CAT_COMMON 1개 제공
- 탑승 중 낙하 데미지 무효화 (rider.fallDistance = 0f)
- 소환 상태(summonedPet/summonedEntityId)는 런타임 전용, NBT 저장 안 됨
- saveAsPassenger() = false로 플레이어 RootVehicle 저장도 차단
- 재접속 시 항상 미소환 상태로 시작 (펫 엔티티 자동 디스폰)
- GUI: 분할 패널 레이아웃 (왼쪽 2열 그리드 + 오른쪽 상세 정보)
- 엔티티 프리뷰: `InventoryScreen.renderEntityInInventoryFollowsMouse()` 사용 (바운딩 박스 기반 자동 센터링)

## 해결한 이슈
- **P키 충돌**: 바닐라 Social Interactions와 충돌 → H키로 변경
- **GUI 엔티티 위치**: `renderEntityInInventory` → `renderEntityInInventoryFollowsMouse`로 교체하여 바운딩 박스 기반 자동 센터링
- **엔티티 뒤집힘**: 수동 좌표 계산 방식에서 자동 센터링 API로 전환하여 해결
- **로그아웃 상태 유지**: `saveAsPassenger() = false` 추가로 RootVehicle 저장 차단
- **소환 후 GUI 재열림**: `handleSummonPet()`에서 `handleRequestStorage()` 호출 제거 (PR 리뷰)
