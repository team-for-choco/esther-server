# 상인 구매/판매 물품 조정 (Phase 11-4)

## 작업 개요
현재 상인 시스템의 구매/판매 가격이 등급별로 일관성이 부족하고, 일부 항목이 누락되어 있음.
기존 가격 체계를 전면 재조정하여 등급별 차등을 명확하게 하고, 구매/판매 비율을 통일한다.

**JIRA**: CHOCO-125

## 작업 전 요청사항
- 강화석 상점 구매: 추가 안 함 (드롭 전용 유지)
- 김/면 구매: 추가 안 함 (현행 유지)
- 커스텀 광물 구매: 추가 안 함 (채광 전용 유지)

## 작업관련 주요 정보

### 설계 원칙
- 수확물 순수익 = 수확물 판매가 - 씨앗 구매가 (씨앗 드롭 없음)
- 농사 순수익은 어종의 ~50-60% (대량 수확 가능하므로)
- 요리 판매가 = 원가 기반 (일반 +5~10, 고급 +15~20, 희귀 배수 1.21~1.3x)
- 광물/수확물: 같은 등급 내 획득률 동일 → 균일 가격
- 어종: 가격 편차에 맞춰 획득 가중치 차등화
- 원석 → 주괴 2x 비율 유지

### 주요 변경사항
1. **씨앗 상인**: 바닐라 씨앗 구매 제거, 물뿌리개 구매(500)/판매(150) 추가
2. **어종**: 등급 내 가격 편차 유지 + WeightedFishLootModifier 가중치 차등화
3. **수확물**: 등급 내 균일화 (일반 13, 고급 33, 희귀 73, 밥 15)
4. **광물**: 등급 내 균일화 (일반 원석 5/주괴 10, 고급 13/26, 희귀 40/80)
5. **요리**: 원가 기반 전면 재조정 (재료 판매가 합산 → 마진 추가)
6. **식재료 상인**: 요리 재료만 남김 (사과/열매/빵/구운대구 제거)
7. **광물 상인**: 철 원석 구매 제거, 레드스톤 구매 제거
8. **특수 상인**: 토지 증서 → 일반 뽑기권(2000)
9. **레드스톤**: 구매 불가, 판매 250 (상자 루트 로또)
10. **보안**: handleBuy quantity 음수 검증 추가

### 수정 파일
- `ShopBuyRegistry.kt` - 구매가 조정
- `ItemPriceRegistry.kt` - 판매가 조정
- `WeightedFishLootModifier.kt` - 어종 획득 가중치 차등화
