# 인챈트 상인 NPC 구현

## 작업 개요

특수 도구 4종의 인챈트 테이블 사용을 차단하고, 화폐로 랜덤 인챈트를 부여하는 새로운 NPC(인챈트 상인)를 구현한다.

CHOCO-143

## 작업 전 요청사항

- 특수 도구 4종(낚싯대/호미/곡괭이/조리도구) 인챈트 테이블 불가 처리
- 새로운 인챈트 상인 NPC 구현
  - 대상: 방어구 등 기존 인챈트 가능 장비 + 특수 도구 4종 전부
  - 인챈트 종류: 모든 종류 (일반적으로 사용 불가한 것 포함)
  - 비용: 화폐(기운)
  - 두 가지 서비스 모드:
    1. **즉시 부여 (저렴)**: 현재 슬롯 수만큼 전부 랜덤으로 교체
    2. **선택 부여 (고가)**: 새 랜덤 인챈트를 미리 보여주고 유저가 적용 여부 선택

## 작업 진행 중 결정된 사항

- 특수 낚싯대는 `getEnchantmentValue()` override가 1.21.4 미지원 → 제거 (기본 Item 반환값 0으로 이미 차단)
- 나머지 3종(호미/곡괭이/조리도구)은 기본 `Item` 이므로 이미 인챈트 테이블 사용 불가
- `EnchantMerchantEntity`: 별도 엔티티 클래스 (MerchantEntity와 독립)
- 손에 든 아이템(mainHandItem)을 대상으로 인챈트 적용
- **비용**
  - 즉시 부여: 500기운
  - 선택 부여: 1500기운 (미리보기 요청 시 차감, 수락 여부 무관)
  - 슬롯 추가: 5000기운
- **슬롯 시스템**
  - 아이템의 현재 인챈트 수 = 슬롯 수 (바닐라 인챈트 포함)
  - 즉시/선택 부여: 현재 슬롯 수만큼 전부 랜덤 교체 (기존 인챈트 초기화)
  - 슬롯 추가: 기존 인챈트 유지하며 새 슬롯 1개 추가 (기존 타입 중복 제외)
  - 최대 슬롯: 4개
  - 슬롯 0개일 때 즉시/선택 부여 비활성화
- 모든 등록된 인챈트를 랜덤 풀로 사용 (데이터팩 포함, 호환성 제한 없음)
- 레벨은 해당 인챈트의 최대 레벨 범위에서 랜덤 선택
- 겹치지 않는 타입으로 슬롯 수만큼 선택
