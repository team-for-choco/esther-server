# 4월 13일 버그 수정 및 개선 사항

## 작업 개요
인게임 테스트에서 발견된 버그 7건을 수정하고, 기능 개선을 진행한다.

## 작업 전 요청사항
- 각 이슈별 Jira 티켓 분리
- 각 이슈별 개별 브랜치로 작업 후 develop으로 PR

## 이슈 목록

### 1. 타인 토지 횃불 설치 시 아이템 미반환
- **유형**: 버그
- **증상**: 다른 플레이어의 토지에 횃불을 설치하면 블록은 사라지지만 아이템이 즉시 돌아오지 않음. 재접속해야 인벤토리에 복구됨.
- **원인**: `ClaimProtectionHandler.onBlockPlace`에서 `EntityPlaceEvent`를 취소하면 블록 상태는 복구되지만, 이미 소비된 아이템은 NeoForge가 자동 복원하지 않음. `sendAllDataToRemote()`로 인벤토리를 동기화하지만, 서버 측에서도 아이템이 이미 차감된 상태라 차감된 인벤토리가 그대로 전송됨.
- **수정 방향**: 이벤트 취소 시 설치된 블록의 아이템을 수동으로 인벤토리에 복원한 뒤 동기화

### 2. 가구(소파) 파괴 시 아이템 미드롭
- **유형**: 버그
- **증상**: 여우 소파를 파괴하면 아이템이 드롭되지 않아 사라짐. 다른 소파도 동일 의심.
- **원인**: `cat_sofa`만 loot table JSON이 존재하고, `fox_sofa`, `dog_sofa`, `rabbit_sofa`에는 loot table이 없음.
  - 더미 블록 쪽에서 파괴 시에는 `Block.popResource()`로 직접 드롭하므로 정상 동작
  - 마스터 블록(본체) 파괴 시에는 Minecraft의 loot table 시스템에 의존 → loot table 부재로 드롭 없음
- **수정 방향**: `fox_sofa.json`, `dog_sofa.json`, `rabbit_sofa.json` loot table 생성

### 3. 4→5레벨 강화 시 강화석 요구량 표시 오류
- **유형**: 버그 (UI + 클라이언트 검증)
- **증상**: 대장장이 NPC 강화 화면에서 4→5 강화 시 강화석이 "x1"로 표시됨. 실제 서버 코드는 5개 요구.
- **원인**:
  - 번역 키 `gui.estherserver.enhancement.requires_stone`에 "강화석 x1 필요"로 하드코딩
  - 클라이언트 `hasEnhancementStone()`이 `any` (1개 이상 존재) 여부만 확인, 수량 미검증
  - 서버 측 `EnhancementHandler`는 정상적으로 5개 검증
- **수정 방향**:
  - 번역 텍스트를 파라미터화: "강화석 x%s 필요"
  - `EnhancementScreen`에서 실제 `stoneCount`를 표시하고, 수량 기반으로 활성화 판단

### 4. 일일 퀘스트 재료를 일반(Common) 등급으로 제한
- **유형**: 개선
- **증상**: 일일 퀘스트에 고급(Advanced) 등급 재료가 포함됨
- **해당 항목**:
  - 낚시: `salmon_catch`(연어), `hairtail`(갈치), `sea_bream`(도미) → Advanced
  - 요리: `dumpling`(만두), `ramen`(라면) → Advanced
- **수정 방향**: 위 5개 항목을 dailyPool에서 제거

### 5. 월요일 00시 야생 자동 리셋 미작동
- **유형**: 버그 (미구현)
- **증상**: 월요일 00시(KST)에 야생이 자동 리셋되어야 하나 동작하지 않음
- **원인**: 스케줄러 코드가 아예 존재하지 않음. `/wild reset` 수동 명령만 구현됨.
- **수정 방향**: 서버 시작 시 스케줄러 등록, KST 기준 매주 월요일 00:00에 `resetWild` 로직 자동 실행

### 6. 야생 리셋 시 네더/엔더 동시 초기화
- **유형**: 기능 추가
- **증상**: 야생 리셋 시 오버월드 야생 차원만 초기화됨
- **현재 상태**: 네더/엔더는 바닐라 차원 사용 중, 커스텀 관리 없음
- **수정 방향**: `/wild reset` 실행 시 네더(`the_nether`)와 엔더(`the_end`) 차원 폴더도 함께 삭제하여 리셋
  - 주의: 네더/엔더에 있는 플레이어도 사전 대피 필요

### 7. 토지 새 청크에 기존 권한 미적용
- **유형**: 버그
- **증상**: `/토지 세팅`으로 권한을 변경한 뒤 새로 설치(claim)하는 청크에는 변경된 권한이 적용되지 않음 (기본값으로 생성됨)
- **원인**: `ChunkClaimManager.claim()`에서 새 `ChunkClaimEntry`를 기본 `ClaimPermissions()`로 생성. 기존 청크의 권한을 참조하지 않음.
- **수정 방향**: 새 청크 claim 시, 플레이어의 기존 클레임이 있으면 해당 권한을 상속
